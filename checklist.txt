check list


Phase 1 : Les Bases

[ ] Comprendre read() : Maîtriser ses 3 valeurs de retour
(octets lus, 0 pour EOF, -1 pour erreur).


[ ] Comprendre static : Saisir qu'une variable static dans une fonction garde 
sa valeur entre les appels (la "mémoire" du GNL).


[ ] Utilitaires : Préparer vos propres fonctions (type strjoin, strchr, substr)
car libft est interdite.





Phase 2 : La Logique

[ ] La "Réserve" (Stash) : Créer une variable static char * pour stocker 
le surplus de lecture.

[ ] Boucle de Lecture : Appeler read en boucle (avec BUFFER_SIZE)  et ajouter 
au stash JUSQU'À ce qu'un \n soit trouvé ou que read retourne 0 ou -1.

[ ] Extraction : Isoler la ligne (jusqu'au \n inclus) depuis le stash.

[ ] Mise à jour : Mettre à jour le stash en ne gardant que ce qui vient après 
le \n retourné.

[ ] Optimisation : Ne pas appeler read si le stash contient déjà une ligne.




Phase 3 : Les Cas Particuliers

[ ] Retour NULL : Retourner NULL en cas d'erreur (read == -1) ou si l'EOF est 
atteint ET que le stash est vide.


[ ] Gestion du \n : La ligne retournée DOIT inclure le \n.

[ ] Gestion de l'EOF : SI le fichier se termine SANS \n, la dernière ligne 
retournée ne doit PAS avoir de \n.

[ ] Gestion Mémoire : Aucune fuite de mémoire. free tout en cas d'erreur.






Phase 4 : Les Tests

[ ] BUFFER_SIZE : Tester avec des valeurs extrêmes (1, 9999, 10000000).

[ ] Compilation : Le Makefile doit compiler avec -D BUFFER_SIZE=n ET aussi sans.


[ ] Stabilité : Aucun crash (segfault, etc.).

[ ] Interdits : Pas de libft , pas de variables globales , pas de lseek.
